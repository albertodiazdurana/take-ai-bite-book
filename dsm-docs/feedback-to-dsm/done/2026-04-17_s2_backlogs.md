## Proposal 1: Sprint-plan structural validation at creation time (Template 8 conformance gate)

**Priority:** Medium
**Complements:** BACKLOG-362 (sprint-boundary-gate-wrap-up)

### Problem

Plans titled "Sprint N — {description}" can be authored and approved using an ad-hoc structure that omits most of DSM Template 8 (`DSM_2.0.C_Sprint_Assessment_Templates.md` §1). Neither `/dsm-backlog` (creates the BL file) nor `/dsm-align` (validates folder scaffold) checks Template 8 conformance. The minimum BL spec in `dsm-docs/plans/README.md` enforces only metadata fields (Status, Priority, Date Created, Origin, Author), not sprint-specific section structure.

BACKLOG-362 addresses the **closure-side** gap: it proposes auto-triggering the Sprint Boundary Checklist at `/dsm-wrap-up` time. But BL-362's checklist-execution assumes the checklist exists *in the plan file*. If the plan was authored without it (the creation-side gap), BL-362 has nothing to trigger against.

### Evidence

- Project: dsm-jupyter-book
- Plan: `dsm-docs/plans/BACKLOG-002_sprint-1-implementation.md`
- Status: Approved end of Session 1 (2026-04-17)
- Template 8 sections missing: Duration, Prerequisites (as distinct section), Research Assessment, Experiment Gate, Branch Strategy (dedicated section), MUST/SHOULD/COULD split, Phases with execution-mode tags, Phase Boundary Checklist (intra-sprint), Open Design Questions, How to Resume, **Sprint Boundary Checklist**
- Sections present: ad-hoc Goal / Inputs / Exit criteria / Work items A-I / Out of scope / Sprint 2 preview / Dependencies / Risks / Notes
- Structural compliance: 0 of 11 Template 8 sections applied
- How this was approved: Gate 1 (collaborative definition) of the 4-gate Pre-Generation Brief Protocol validated substantive scope (work items cover the capability) but included no structural check against Template 8

### Proposal

Add a creation-time structural gate. Two possible implementation points:

**Option A:** `/dsm-backlog` detects sprint-plan intent via title regex `^Sprint \d+\b` (or a `--sprint` flag) and inserts the Template 8 skeleton when the BL file is created. Author fills in the sections rather than inventing a structure.

**Option B:** `/dsm-align` adds a lint check: for each file in `dsm-docs/plans/` whose title matches `^Sprint \d+\b`, verify the presence of Template 8 section headings. Report missing sections as warnings in the post-change report.

Option A is preventive (catches at creation). Option B is detective (catches drift on already-created plans). Both are cheap and complementary.

### Relation to existing BLs

| BL | Lifecycle moment | Gate |
|----|------------------|------|
| This proposal | Plan creation | Template 8 skeleton / lint |
| BACKLOG-362 | Sprint closure | Auto-trigger of Sprint Boundary Checklist |
| BACKLOG-363 | Verification | Checkbox reconciliation |
| BACKLOG-364 | Epoch close | Epoch Boundary Checklist |

The four form a plan-lifecycle validation chain: creation → execution → closure → epoch rollup. This proposal fills the "creation" gap.

### Risks

- Template 8 skeleton insertion may feel heavy for trivial one-item "sprints"; mitigation: make the skeleton optional (prompt at `/dsm-backlog` time) or drive it off the `--sprint` flag.
- False positives in the lint check for sprint-adjacent BLs (e.g., "Sprint 1 preparation") that aren't full sprint plans; mitigation: tighten the regex or require explicit opt-in via a front-matter field.

## Proposal 2: `dsm-docs/plans/README.md` authoring-guide pointer

**Priority:** Low

### Problem

`dsm-docs/plans/README.md` currently lists BL filename format and required metadata fields but does not point authors to Template 8 when the BL is a sprint plan. Template 8 is authoritative in `DSM_2.0.C_Sprint_Assessment_Templates.md` §1 but is not discoverable from the plans folder's own README.

### Evidence

See Proposal 1 evidence. BL-002 author (this project) consulted `plans/README.md` for format guidance and received minimum-BL guidance, not sprint-BL guidance.

### Proposal

Add one line to the template `plans/README.md` generated by `/dsm-align`:

> When creating a Sprint-N BL (title matches "Sprint N — ..."), use DSM Template 8 (`DSM_2.0.C_Sprint_Assessment_Templates.md` §1) for the plan structure. The minimum fields above are in addition to, not instead of, Template 8 sections.

Low-cost documentation fix that reduces the class of errors Proposal 1's tooling would also catch. The two are independent and together cover the gap from both angles (documentation + tooling).
